Install skillctl

Get onto __SKILLCTL_VERSION__ — the version that makes cross-person trust real.

1

Install / update to __SKILLCTL_VERSION__

Your first build is v0.2.2; the secure sign/verify path only exists in __SKILLCTL_VERSION__.

REQUIRED
curl -fsSL https://github.com/kamir/m3c-tools/releases/download/skillctl/__SKILLCTL_VERSION__/install.sh | RELEASE_BASE=https://github.com/kamir/m3c-tools/releases/download/skillctl/__SKILLCTL_VERSION__ bash
hash -r                 # clear the shell's cached path to any OLD skillctl
skillctl version        # MUST print __SKILLCTL_VERSION__
which skillctl          # MUST be ~/.local/bin/skillctl
Expected: skillctl __SKILLCTL_VERSION__ (installer verifies the release signature, fp 5f8f39cb…). ⚠ If version shows an OLDER version, or a command like login is "unknown command": a STALE skillctl is ahead on your PATH (often ~/go/bin/skillctl). Fix: hash -r, put ~/.local/bin FIRST on PATH (step 2), or remove the old one (rm ~/go/bin/skillctl) — then re-check which skillctl + skillctl version.
2

PATH check — make sure the NEW skillctl wins

"command not found" OR an old build answering (wrong version / "unknown command") both mean PATH: put ~/.local/bin first.

if needed
export PATH="$HOME/.local/bin:$PATH"   # ~/.local/bin FIRST; add to ~/.zshrc to persist
hash -r
which skillctl       # → /Users/<you>/.local/bin/skillctl
skillctl version     # → __SKILLCTL_VERSION__
Tip: a quick bulletproof alternative is to call it by full path — ~/.local/bin/skillctl … — which ignores PATH entirely.
3

Package a skill & release

Key → skill → publish (packs + signs + shares) → attest.

3

Generate your signing key

One ed25519 keypair. .priv never leaves your Mac; only .pub is shared.

REQUIRED
skillctl keygen --out {{keystem}}
Produces {{keystem}}.priv (0600) and {{keystem}}.pub. Already have a key? keygen refuses to overwrite — that's expected: reuse your existing key, skip to step 4 (don't rotate unless you mean to).
4

Read your fingerprint to Kamir 🔑

The load-bearing trust step. Kamir pins THIS value, or your skill won't verify on his side.

REQUIRED
openssl pkey -pubin -in {{keystem}}.pub -outform DER | tail -c 32 | shasum -a 256
Read the sha256 aloud on the call; Kamir re-derives it and confirms.
5

Get the skill into ~/.claude/skills/

A skill is a folder with a SKILL.md (name = folder, semver version, ≥20-char description, governance_level green).

REQUIRED
mkdir -p ~/.claude/skills
cp -R ./{{skill}} ~/.claude/skills/{{skill}}
ls ~/.claude/skills/{{skill}}/SKILL.md
# optional self-check — only runs if the skill ships one:
[ -f ~/.claude/skills/{{skill}}/tests/smoke.sh ] && bash ~/.claude/skills/{{skill}}/tests/smoke.sh || echo "(no tests/smoke.sh — optional, skip)"
SKILL.md must exist. A tests/smoke.sh is optional — publishing does not require it.
6

Authenticate — skillctl login

From __SKILLCTL_VERSION__, skillctl logs in via the browser and uses the device token automatically (FR-0043) — no manual export, no minted token.

REQUIRED
skillctl login --base-url https://onboarding.guide   # browser -> public SaaS; saves the device token
skillctl login --status                              # should print: Logged in
⚠ Use --base-url https://onboarding.guide (the public SaaS) — without it, older builds default to the local server and login fails. Headless/SSH? add --no-browser and open the printed URL. Last-resort fallback: export ER1_DEVICE_TOKEN=<token from Kamir>.
7

Publish & share (packs → signs → posts)

One command packs the dir into a .skb, signs it with your key, admits it to your context, and maps it into the room.

REQUIRED
cd ~
skillctl publish {{skill}}@{{ver}} \
  --skill-dir ~/.claude/skills/{{skill}} \
  --registry self --er1-target prod \
  --er1-context {{ctx}} \
  --identity {{identity}} \
  --key {{keystem}}.priv \
  --share-room {{room}}
Produces ~/{{skill}}@{{ver}}.skb. Copy the sha256 digest it prints into the output box for Kamir.
⚠ 403 "Not authorized for this context"? You can only publish into your own context. The --er1-context sub-prefix must match the Google account you logged in with (step 6). Logged in as someone else (e.g. testing with another account) → use your own …___skills context + matching --identity.
8

Attest it green

Self-attest = peer trust (fine for our room). Lets Kamir's governance gate pass.

REQUIRED
skillctl publish --attest {{skill}}@{{ver}} --level green \
  --rationale 'author self-attest, {{room}} peer trust' \
  --er1-target prod --er1-context {{ctx}} \
  --identity {{identity}} --key {{keystem}}.priv
9

Hand the public key to Kamir

Send the .pub; the fingerprint (step 4) makes it trustworthy.

REQUIRED
cat {{keystem}}.pub      # send this public key to Kamir
4

Upgrade the bundle to next version & release

Bump → re-validate → publish {{skill}}@{{nextver}} → attest.

10

Bump the version in the skill

The registry enforces version > last admitted, so {{nextver}} must be higher than {{ver}}.

REQUIRED
sed -i '' 's/^version: .*/version: {{nextver}}/' ~/.claude/skills/{{skill}}/SKILL.md
grep '^version:' ~/.claude/skills/{{skill}}/SKILL.md
Expect version: {{nextver}}. (Edit the SKILL.md body too if the skill itself changed.)
11

Re-validate the skill (optional)

If the skill ships a smoke test, run it on the bumped version. Many skills have none — that's fine.

optional
[ -f ~/.claude/skills/{{skill}}/tests/smoke.sh ] && bash ~/.claude/skills/{{skill}}/tests/smoke.sh || echo "(no tests/smoke.sh — optional, skip)"
No tests/smoke.sh? Skip — it's optional. At minimum re-check SKILL.md still has version: {{nextver}}.
12

Publish the next version

Packs + signs a NEW bundle {{skill}}@{{nextver}}.skb and shares it into the room.

REQUIRED
cd ~
skillctl publish {{skill}}@{{nextver}} \
  --skill-dir ~/.claude/skills/{{skill}} \
  --registry self --er1-target prod \
  --er1-context {{ctx}} \
  --identity {{identity}} \
  --key {{keystem}}.priv \
  --share-room {{room}}
Copy the new digest into the output box — it differs from {{ver}}.
13

Attest {{nextver}} green & notify Kamir

Attest the new digest; tell Kamir the new version + digest so he can re-pull.

REQUIRED
skillctl publish --attest {{skill}}@{{nextver}} --level green \
  --rationale 'v{{nextver}} self-attest, {{room}} peer trust' \
  --er1-target prod --er1-context {{ctx}} \
  --identity {{identity}} --key {{keystem}}.priv
