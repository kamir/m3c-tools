What is "Multimodal Memory Capturing"?

A desktop-native system that captures observations about digital content through multiple sensory channels and feeds them into an ER1 knowledge server for persistent, searchable, AI-augmented memory. Each capture combines up to three modalities into a single memory entry.

📝
Text / Transcript
YouTube captions, composite documents,
user annotations. Always present.
🎙️
Voice Impression
16kHz mono WAV, transcribed via Whisper.
Captures subjective reaction & context.
📸
Screenshot
PNG screen capture or clipboard image.
Visual context for the observation.

Data Flow: Desktop → ER1

┌─────────────────────────────────────────────────────────────────┐ │ DESKTOP CAPTURE LAYER (macOS Menu Bar App — Go/menuet) │ │ │ │ [Fetch Transcript] → YouTube InnerTube API → Caption XML │ │ [Record Impression] → PortAudio mic capture → WAV (16kHz) │ │ [Screenshot] → screencapture CLI → PNG │ │ [Audio Import] → Scan GDrive folder → MP3/WAV │ │ │ │ Whisper STT: audio → text transcription (subprocess) │ └────────────────────────────┬────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────────────────┐ │ LOCAL MEMORY LAYER ~/Library/Application Support/YTTranscript │ │ │ │ MEMORY/MEMORY-{timestamp}/ │ │ ├── transcript_{ts}.txt # composite document (always) │ │ ├── impression_{ts}.wav # voice recording (optional) │ │ ├── screenshot_{ts}.png # screen capture (optional) │ │ ├── tag.txt # tags, one per line │ │ └── metadata.json # structured metadata │ │ er1_retry_queue.json # persistent retry state │ │ audio_tracker.txt # batch import tracker │ └────────────────────────────┬────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────────────────┐ │ ER1 UPLOAD LAYER (offline-first, retry with backoff) │ │ │ │ POST /upload_2 (multipart/form-data) │ │ Headers: X-API-KEY: {api_key} │ │ Files: transcript_file_ext (.txt) ← always │ │ audio_data_ext (.wav) ← optional │ │ image_data (.png) ← optional │ │ Data: context_id, content_type, tags │ │ │ │ Retry: exponential backoff (30s → 1h), max 10 attempts │ │ Queue: persistent JSON file, survives app restart │ └────────────────────────────┬────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────────────────┐ │ ER1 SERVER (Flask + GCS + Firestore) │ │ │ │ → Audio → gs://er1-stage/uploads/{context}_{ts}.mp3 │ │ → Image → gs://er1-stage/uploads/{context}_{ts}.jpeg │ │ → Text → Firestore: smart-audio-transcriptions/{doc_id} │ │ → Tags → Firestore document fields │ └─────────────────────────────────────────────────────────────────┘

Workaround: send 1x1 transparent PNG placeholder. ── Response (200) ── { "doc_id": "hdadDimjhDjMqygIkBmD", "collection_name": "smart-audio-transcriptions", "gcs_uri": "gs://er1-stage/uploads/...mp3", "gcs_uri_img": "gs://er1-stage/uploads/...jpeg", "transcript": "(transcript text)", "time": "2026-03-09 21:30:01" }
